# Data

This project uses one default public-safe SA-CCR sample dataset.

## Default Sample Portfolio

`sample_trades.csv` is a deterministic, anonymized, industry-style derivative portfolio generated by:

```bash
python scripts/build_industry_style_sample.py
```

The sample is designed to resemble bank SA-CCR implementation templates at the field and product-coverage level. It does not copy client/project trade rows, original counterparty names, or confidential source data.

The generated dataset includes 298 trades across 5 SA-CCR asset classes and 19 product types:

- FX: FX Forward, FX Swap, FX Option
- Interest Rate: Interest Rate Swap, Cross Currency Swap, FRA, OIS, Swaption, Cap/Floor
- Commodity: Precious Metal Forward, Precious Metal Swap, Commodity Option, Oil Forward, Electricity Swap
- Credit: Credit TRS, Credit Single Name CDS, Credit Index CDS
- Equity: Equity TRS, Equity Index Option

## Files

- `sample_trades.csv`: trade-level product economics and SA-CCR input fields
- `counterparty_reference.csv`: synthetic counterparty type, rating, and illustrative risk weight
- `collateral.csv`: synthetic netting-set level collateral and CSA-style fields

## Field Design

The trade file includes product economics and internal bank fields needed for a full SA-CCR demonstration:

- product type, asset class, subclass, currency pair, underlier, maturity date, start date, and end date
- leg notionals and USD-equivalent base notional
- margin flag, CCP flag, margin frequency, counterparty id, and netting set id
- buy/sell direction, option put/call flag, strike, underlying price, and exercise date
- MtM and synthetic source identifiers

`collateral.csv` is netting-set level synthetic CSA data and includes:

- `Margin Agreement ID`
- `Collateral Amount`
- `Threshold Amount`
- `Minimum Transfer Amount`
- `Net Independent Collateral Amount`

These fields support the margined replacement cost formula `max(V-C, TH+MTA-NICA, 0)` and the unmargined EAD cap calculation used for margined netting sets.

## Why Synthetic Data Is Used

SA-CCR requires internal bank information that is normally not available in public transaction feeds, including legal netting sets, CSA terms, collateral balances, MtM, trade direction, internal counterparty mapping, and regulatory risk weights.

Using deterministic synthetic data lets the project be:

- safe to publish on GitHub
- reproducible for interview review
- broad enough to demonstrate all five SA-CCR asset classes
- clear about which assumptions are illustrative

Regenerate the files at any time with:

```bash
python scripts/build_industry_style_sample.py
```
