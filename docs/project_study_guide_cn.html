2. 项目一句话说明

这个项目是一个基于 Python 的 Basel-aligned SA-CCR demo engine。它读取 synthetic derivative portfolio， 对每笔交易做 SA-CCR enrichment，然后按 counterparty、netting set、asset class、hedging set 聚合， 最终输出 RC、PFE、EAD 和 illustrative RWA，并生成 HTML 报告和 Excel 对账表。

I built a Python-based Basel-aligned SA-CCR exposure engine covering trade enrichment, adjusted notional, supervisory delta, maturity factor, asset-class add-on aggregation, RC/PFE/EAD calculation, margined EAD cap logic, and reviewer-facing HTML/Excel outputs.

5. Trade-Level Enrichment

src/saccr_engine/enrichment.py

这是最关键的计算模块之一。它把原始交易字段转换成 SA-CCR 需要的逐笔计算字段。 最重要的输出是 Effective Notional。

Effective Notional = Adjusted Notional * Delta * Maturity Factor
字段含义
Remaining Maturity Years到期日到 COB date 的剩余期限。
Maturity BucketIR 使用的 <1Y、1Y-5Y、>5Y。
Hedging SetIR 按 currency，FX 按 currency pair，credit/equity 按 entity，commodity 按 group。
Supervisory FactorBase factor 乘以 basis/volatility adjustment。
Supervisory DurationInterest Rate 和 Credit 的 duration adjustment。
Delta线性交易为 +/-1；option 使用 supervisory option delta。
Maturity FactorUnmargined 用 remaining maturity；margined 用 MPOR。

Supervisory Duration

SD = (exp(-0.05 * S) - exp(-0.05 * E)) / 0.05

只对 Interest Rate 和 Credit 使用。其他 asset class 当前设为 1。

Option Delta

Call delta = direction * N(d1) Put delta = -direction * N(-d1) d1 = [ln(P / K) + 0.5 * sigma^2 * T] / [sigma * sqrt(T)]

Maturity Factor

Unmargined
MF = sqrt(min(M, 1))
Margined
MF = 1.5 * sqrt(MPOR / 250)

6. AddOn 聚合

src/saccr_engine/addon.py

AddOn 模块把逐笔交易的 effective notional 按 asset class 的规则聚合成 netting set add-on。 不同 asset class 的 aggregation 逻辑不一样。

Asset Class聚合逻辑要点
Interest Rate 按 currency hedging set 和 maturity bucket 聚合。 使用 D1/D2/D3 bucket partial offset formula。
FX 按 currency pair 聚合。 同一 pair 内 long/short 可以抵消。
Credit / Equity 先按 underlying entity，再做 correlation aggregation。 Index 和 single-name correlation 不同。
Commodity 按 commodity group/type 做 correlation aggregation。 Electricity 使用单独 factor 和 volatility bucket。

Interest Rate 公式

Aggregated Effective Notional = sqrt(D1^2 + D2^2 + D3^2 + 1.4*D1*D2 + 1.4*D2*D3 + 0.6*D1*D3) IR AddOn = Aggregated Effective Notional * 0.005

Credit / Equity / Commodity correlation 公式

Entity AddOn = Effective Notional * Supervisory Factor Systematic Component = Correlation * Entity AddOn Idiosyncratic Component = (1 - Correlation^2) * Entity AddOn^2 AddOn = sqrt(Systematic_Total^2 + Idiosyncratic_Total)
面试表达：Add-on aggregation is asset-class specific. The final netting-set AddOn is the sum of asset-class add-ons.

7. RC / PFE / EAD

src/saccr_engine/exposure.py

Exposure 模块把 netting-set AddOn 转成监管暴露结果：RC、PFE、EAD、RWA。

Unmargined RC
RC = max(V - C, 0)
Margined RC
RC = max(V - C, TH + MTA - NICA, 0)
EAD
EAD = 1.4 * (RC + PFE)

PFE multiplier

multiplier = min(1, floor + (1 - floor) * exp((V - C) / [2 * (1 - floor) * AddOn])) floor = 0.05 PFE = multiplier * AddOn Aggregate

直觉：如果 netting set 是 over-collateralized 或 negative MtM，multiplier 会下降； 但不会低于 5% floor。

Margined EAD Cap

Margined EAD <= same netting set calculated on an unmargined basis

项目会强制把同一 portfolio 按 unmargined 再跑一次。如果 margined EAD 高于 unmargined basis EAD， 则把它 cap 到 unmargined EAD。

11. 面试问答模板

Q1: 你这个项目主要做什么？

This project implements a Basel-aligned SA-CCR portfolio exposure workflow in Python. It starts from synthetic trade, collateral, and counterparty reference data, enriches trades with SA-CCR fields, aggregates add-ons by asset class and netting set, calculates RC, PFE, EAD, and illustrative RWA, and produces both an HTML report and an Excel reconciliation workbook.

Q2: 为什么不用 notebook？

I wanted this to look like a maintainable Python project rather than a one-off notebook. The code is split into modules for configuration, data checks, enrichment, add-on aggregation, exposure calculation, pipeline orchestration, and reporting.

Q3: SA-CCR 的核心公式是什么？
EAD = alpha * (RC + PFE) alpha = 1.4 PFE = multiplier * AddOn Aggregate
Q4: Excel 对账表的作用是什么？

The Excel workbook is an independent transparency layer. It recreates the main Python calculations in Excel formulas and reconciles trade-level effective notional, add-on detail, netting-set EAD, and counterparty EAD against the Python outputs.

Q5: 这个项目和真实监管系统有什么差距？

It is Basel-aligned for the core SA-CCR exposure workflow, but it is not a production regulatory capital engine. It simplifies legal netting enforceability, collateral eligibility and haircuts, product taxonomy, market data sourcing, risk-weight calculation, CVA/CCP capital, and formal model governance.

13. 重点记住的 8 句话

  1. 这个项目是 Basel-aligned SA-CCR core exposure workflow demo，不是完整生产监管系统。
  2. 核心公式是 EAD = 1.4 * (RC + PFE)。
  3. PFE = multiplier * AddOn Aggregate。
  4. 逐笔交易最核心字段是 Effective Notional = Adjusted Notional * Delta * Maturity Factor。
  5. AddOn aggregation 是 asset-class specific 的。
  6. RC 是当前暴露，PFE 是潜在未来暴露。
  7. Margined netting set 使用 CSA 字段计算 RC，并应用 unmargined EAD cap。
  8. Excel reconciliation workbook 用公式复刻 Python 结果，是 reviewer-facing audit layer。
